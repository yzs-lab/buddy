收敛完成

双方独立确认「完成」才会停止。不会因为一方过早判定而遗漏问题。Actor 完成后自动切换至对手方，你随时可以介入校准。

若一方说完成、另一方却继续改了代码，break 会被撤回，任务继续运行——收敛必须建立在双方都无新改动之上。

新手指南

新用户指南

Buddy 是一个桌面应用（macOS），让两个 AI 编程工具围绕你的本地代码仓库轮流工作。一个负责写代码，一个负责检查，交替进行，直到双方都认为任务完成。

使用前必须准备

Buddy 本身不内置 AI 能力。你需要先在自己的电脑上安装准备使用的 AI 命令行工具，并确认这些工具已经登录、授权可用。默认双 AI 协作建议先装好 Claude Code 和 Codex CLI。

Claude Code — Anthropic 出品的编程命令行工具 Codex CLI — OpenAI 出品的编程命令行工具 OpenCode — 开源编程命令行工具 Kimi Code — 月之暗面出品的编程命令行工具

安装后，先在终端里运行一次并确认能正常使用。只检查版本号不够；如果没有登录、登录态过期或权限不足，Buddy 也无法替你调用它。

基本用法

1

安装并登录 AI 命令行工具

先安装你想用的 AI 工具，比如 Claude Code 或 Codex。安装后在终端里运行一次，确认已经登录且能正常使用。

2

配置启动命令

打开 Buddy 的设置，填写 AI 工具的启动命令。默认配置通常可以直接用，也可以自定义路径和参数。

3

选择本地仓库

在 Buddy 中选择一个本地 Git 仓库作为工作目录。两个 Actor 都会在这个目录里读写代码。

4

写任务说明，开始运行

写清楚你想要什么，然后点开始。执行者先写代码，完成后审查者检查，交替进行直到双方都确认完成。

FAQ

常见问题

Buddy 和单个 AI CLI 工具有什么区别？

Buddy 让两个 AI Actor（执行方 + 审查方）轮流工作：执行方写代码，审查方挑毛病，循环推进。这比单次对话更接近真实团队协作，能显著提升一次通过率与代码质量。

任务什么时候会结束？

采用「双确认结束」机制：执行方和审查方都必须发出 type=break 信号，任务才会转为 DONE。单方 break 不会终止任务，避免一方偷懒提前结束。

崩溃或重启后任务还在吗？

在。所有任务状态以文件形式持久化在 ~/Library/Application Support/buddy/。重启时，处于 RUNNING 的任务会自动重置为 PAUSED，方便你继续。

能边运行边追加指令吗？

可以。Actor 运行期间发送的指令会进入指令队列，当前轮次结束后自动执行，无需打断。

数据会离开本地吗？

Buddy 本身不联网传输你的任务数据，所有状态、对话记录、产物都保存在本地文件系统。AI 推理由你配置的 CLI 工具各自完成。
